# 任务产物：AgentExecutionPanel 后端 API 集成

## 任务目标
修复 AgentExecutionPanel.tsx 的 `handleStart` 函数，让它真正调用后端 Agent API (5002)，而不是纯前端模拟。

## 关键发现
1. 原代码的 `handleStart` 是纯前端模拟，没有调用任何后端 API
2. AgentExecutionPanel 导入了 `API_ENDPOINTS` 配置，但从未使用
3. 后端 Agent API (5002) 支持 `/api/agent/process` 端点，可以处理中文/英文任务指令

## 修复内容

### 文件: `web-app/src/components/AgentExecutionPanel.tsx`

**修改前**：
- `handleStart` 函数内部使用 `for` 循环 + `setTimeout` 模拟执行
- Harness 检查硬编码为 `{ allowed: true }`，没有调用后端

**修改后**：
```typescript
const response = await fetch(API_ENDPOINTS.agent.process, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ command })
})
const agentResult = await response.json()
// 显示后端返回的 Harness 约束结果
if (agentResult.harness_constraints) {
  setHarnessResult(...)
}
```

## 数据流

```
用户点击"开始执行"
  ↓
handleStart() 捕获任务模板
  ↓
转换成自然语言指令："在东墙钻一个孔"
  ↓
POST /api/agent/process { command: "在东墙钻一个孔" }
  ↓
后端 Agent API 处理（关键词匹配 + 规则引擎）
  ↓
返回 { result, safe, reason, steps, harness_constraints }
  ↓
前端显示 Harness 约束结果 + 执行步骤
```

## 后端 API 响应格式
```json
{
  "result": "success",
  "safe": true,
  "reason": "东墙为非承重墙，允许钻孔",
  "steps": [
    { "id": 1, "label": "移动到钻孔位置", "action": "move_to_drill_pos", "duration_s": 5 },
    { "id": 2, "label": "检测墙面材质", "action": "detect_wall_material", "duration_s": 3 },
    { "id": 3, "label": "钻孔", "action": "drill_hole", "duration_s": 10 }
  ],
  "harness_constraints": [
    { "type": "承重墙", "passed": true, "message": "东墙为非承重墙 ✅" }
  ]
}
```
