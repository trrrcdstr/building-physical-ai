## 3DGS可行性分析

### 现有VR用于3DGS的评估

**问题：**
1. ❌ **无恒大云东海项目VR** — 当前数据库完全不覆盖佛山地区
2. ⚠️ **720yun/Justeasy/3d66平台VR** — 均为渲染全景图，非真实拍摄，3DGS重建会引入渲染artifacts
3. ⚠️ **视角有限** — 大部分VR是单点360°全景，缺少空间位移的多视角数据
4. ⚠️ **分辨率未知** — 需要确认各平台VR的实际分辨率

**可用于3DGS的前提条件：**
- 同一空间有≥3个不同机位的全景图
- 全景图分辨率≥4K（理想8K）
- 场景内相机位姿可估计

### Index 8 的3DGS潜力

Index 8（Justeasy, 14间, 1752浏览量）最有潜力：
- ✅ 14个房间 = 14个不同视角的全景数据
- ✅ 别墅/叠排户型，与恒大云东海产品类型匹配
- ⚠️ 需要验证各房间全景是否属于同一套房
- ⚠️ 需要确认全景分辨率和提取方式

3. **房产平台**：
   - 贝壳找房/链家VR看房（可能包含恒大云东海二手房）
   - 安居客/房天下VR样板间

### 优先级2：自行拍摄/采集

如果线上找不到恒大云东海VR，考虑：

1. **实地拍摄**（推荐）：
   - 使用Insta360/Ricoh Theta等360°相机实地拍摄
   - 每个房间至少3个机位，覆盖整个户型
   - 拍摄间距约1-2米，确保3DGS有足够overlap
   - 预算：360°相机（3000-8000元）+ 拍摄时间

2. **无人机航拍外景**：
   - 使用DJI Mini等无人机拍摄小区外景
   - 多角度环绕拍摄，用于建筑外观3DGS

### 优先级3：利用现有库做3DGS验证

1. 先用 Index 8（14间别墅VR）验证3DGS pipeline

3. 搭建自动化VR→3DGS训练流程
