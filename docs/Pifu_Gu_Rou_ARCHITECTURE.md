## 一、核心理念：皮·骨·肉三重奏

| 层级 | 数据来源 | 技术实现 | 机器人获得什么 |
|------|----------|----------|----------------|
| **皮（外观）** | VR效果图、实景全景 | 3DGS实景重建 → 1:1高保真视觉复刻 | "这房间长什么样" |
| **骨（结构）** | CAD施工图、机电图 | 几何解析 → 边界、承重、管线拓扑 | "墙在哪、能碰吗、里面有什么" |
| **肉（常识）** | 材料表、建筑规范、物理规则 | Harness层 → 物理参数绑定+因果推理 | "玻璃湿滑需降速、承重墙禁止钻孔" |

**协同关系**：皮让人看见世界，骨让人理解边界，肉让人安全干活。

```
[用户点击"住宅/商场/园林"场景卡片]
```

## 四、3DGS + 3D AIGC 技术分工

| 技术 | 负责什么 | 在项目中的角色 |
|------|----------|----------------|
| 3DGS（高斯溅射） | 实景复刻 | 将VR实景图/全景图重建为1:1高保真3D场景（"皮"的来源） |
| 3D AIGC | 场景生成 | 当没有真实VR时，基于CAD+语义描述生成合理的场景外观（补充"皮"） |

**协同策略**：
- 有VR实景: VR视频 → 3DGS重建 → 照片级3D场景
- 无VR实景: CAD语义 + 3D AIGC → 生成式3D场景
- 半实景: 实景局部3DGS + AIGC补全未覆盖区域

## 六、落地路径（分阶段）

### Phase 1: 场景调度引擎（当前可做）
- 场景卡片组件：按6类场景切换数据源
- 统一数据接口：每个场景类型对应一套皮/骨/肉数据
- 前端路由：点击场景卡片 → 加载对应3D场景+物理面板数据

### Phase 2: 皮·骨·肉数据绑定
- 皮：VR效果图 → 已有1043张渲染图 + 444条VR链接
- 骨：CAD几何 → 151个建筑对象（门/窗/墙体）
- 肉：物理常识 → 22种材质参数 + 箭牌卫浴612产品编码 + 睿研灵巧手6产品线

### Phase 3: 交互增强
- 点击物体 → 物理属性弹窗（材质/重量/摩擦/安全约束）
- Agent指令 → 自动检查物理约束（Harness层）
- 3DGS实时渲染集成
